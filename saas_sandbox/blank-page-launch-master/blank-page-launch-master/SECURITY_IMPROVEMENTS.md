# 安全性改進實施報告

## 概述
本次安全性改進專注於封堵公開端點濫用風險、收斂敏感資料存取權限，並提升整體 Web 安全基線。

## 已實施的安全措施

### 1. Edge Function 安全強化 ✅

#### CORS 白名單機制
- **問題**: 原本使用 `Access-Control-Allow-Origin: *` 允許任何來源存取
- **解決方案**: 實施動態 CORS 白名單，僅允許授權域名
- **允許的域名**:
  - 正式環境: `https://gpt-5-store-keywords.lovable.app`
  - 預覽環境: `https://gpt-5-store-keywords-*.lovable.app`
  - 本地開發: `http://localhost:5173`, `http://localhost:3000`

#### 來源驗證與封鎖
- 新增 `isOriginAllowed()` 函數驗證請求來源
- 未授權來源將收到 403 錯誤回應
- 記錄可疑請求以便監控

#### 輸入驗證與長度限制
- **商店名稱**: 最多 100 字元
- **地址**: 最多 200 字元  
- **描述**: 最多 500 字元
- **關鍵字**: 最多 20 個，每個最多 50 字元
- **自訂感受**: 最多 10 個，每個最多 50 字元

#### 日誌安全性改善
- 生產環境減少敏感資料記錄
- 新增錯誤 ID 便於支援追蹤
- 避免記錄使用者輸入內容

### 2. 速率限制機制 ✅

#### 資料庫表格
- 建立 `function_usage_logs` 表追蹤使用狀況
- 以來源域名為主、IP 位址為輔的識別機制
- 自動清理 24 小時前的記錄

#### 限制規則
- **generate-review 函數**: 15 分鐘內最多 20 次請求
- **超限封鎖**: 超過限制後封鎖 60 分鐘
- **友善錯誤**: 提供剩餘次數與重置時間資訊

#### 錯誤處理
- 資料庫錯誤時允許請求通過，避免影響正常服務
- HTTP 429 狀態碼與標準 `Retry-After` 標頭

### 3. 資料庫權限收斂 ✅

#### 管理員表格 (admins)
- **原狀況**: 允許所有人讀取
- **改善**: 僅超級管理員可存取

#### 監控表格 (monitoring_alerts, monitoring_metrics)
- **原狀況**: 允許所有人讀取
- **改善**: 僅超級管理員可存取

#### 商店關鍵字 (store_keywords)
- **原狀況**: 允許所有人讀取所有關鍵字
- **改善**: 匿名使用者僅能查看有效訂閱商店的關鍵字

### 4. Web 安全標頭強化 ✅

#### 新增的安全標頭
- **Content-Security-Policy**: 防止 XSS 攻擊
- **Strict-Transport-Security**: 強制 HTTPS 連線
- **Referrer-Policy**: 控制 Referrer 資訊洩漏
- **Permissions-Policy**: 限制瀏覽器功能存取
